Section 3
Open Questions
Still pending. Each one blocks something in the roadmap — resolve before building the affected feature.
HUD pills: persistent vs tap-to-expand?
Current build: tap a joint to expand a popup with scores. Spec called for 3 persistent floating pills (shoulder · hip · knee) always visible during playback. Need to see both options side-by-side before deciding. Blocks: Video tab final design
Pony Club performance task list
Dressage tasks are mapped (DECISIONS.md Mar 12). The Pony Club D/C/B/A skills list needs to be sourced and entered. Who owns this content? Blocks: Phase 2 Pony Club
Report tab: position scores AND The Scales, or just The Scales?
Both layers are useful at the ride level. Currently the tab shows position scores only. Should riding quality (The Scales) also appear in Report, or is that only for Progress? Blocks: Report tab final design
Section 4
Decision Log
Recent decisions from our working sessions. Full historical log is in _agents/DECISIONS.md.
2026-03-29
3-tab nav — Home removed, Rides is landing page
Bottom nav simplified to Rides · Progress · Journey. Record action lives inside Rides. Home removed — add back only if user testing shows the need. Supersedes March 11 "4-tab nav" decision.
2026-03-29
"Insights" renamed "Progress"
The aggregate trends tab is now called Progress. The word "Insights" is retired — it was doing three different jobs.
2026-03-29
Analysis sandbox wired into Ride Detail
src/pages/analysis/ becomes the Video + Report tabs inside /rides/:id. RidesPage.tsx and RideDetailPage.tsx must not be touched — add tabs as a new layer.
2026-03-29
Body navigation pinned to Phase 4
Post-ride body review stays scroll-based for now. Body-as-navigation is a Phase 4 display-layer swap. The Phase 1 zone cards must be structured correctly (one card per zone) so the swap is mechanical in Phase 4.
2026-03-29
Discipline selection required for Journey
Journey content is discipline-specific. MVP: Dressage + Pony Club. User selects in onboarding or Settings. Journey shows a prompt if discipline not set. Display labels are discipline-aware: "movements" for dressage, "skills" for pony club.
2026-03-29
Progress chart time axis fix required
Current bar charts have no time axis — unreadable as trend data. Must label chronologically: left = oldest, right = most recent.
2026-03-29
Ghost Rider: toggle (opt-in), not always-on
The ideal posture ghost overlay is a toggleable button on the video frame. Opt-in is cleaner and less distracting. Can revisit to make default-on later.

Commit to Repo
Run these commands in Terminal to save the product plan and roadmap into the horsera_app repo permanently. Replace the path with your actual repo location.
# 1. Navigate to your repo
cd ~/Documents/Horsera/horsera_app

# 2. Copy the downloaded files into _product-docs/
cp ~/Downloads/HORSERA_PRODUCT_PLAN.md _product-docs/
cp ~/Downloads/HORSERA_ROADMAP.html _product-docs/

# 3. Stage the files
git add _product-docs/HORSERA_PRODUCT_PLAN.md _product-docs/HORSERA_ROADMAP.html

# 4. Commit
git commit -m "docs: add product plan and visual roadmap (2026-03-29)"

# 5. Push
git push origin main

Not sure of your repo path? Open Terminal and type cd ~/ then drag the horsera_app folder from Finder into the Terminal window — it'll paste the full path automatically.
